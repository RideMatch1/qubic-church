## PART C: Probability Critique

### The Multiple Comparisons Problem

The original document claimed P < 10^-80 for all findings. This is **statistically misleading** because:

1. **Post-hoc analysis:** Probabilities were calculated AFTER finding patterns, not predicted beforehand
2. **Cherry-picking:** Only correlations that "worked" are reported; failures are ignored
3. **No correction:** Multiple comparisons require Bonferroni correction

**Example of the fallacy:**
- Looking at 1,000 Bitcoin addresses for ANY pattern will find "interesting" things
- Finding "1CFB" THEN calculating its probability is circular reasoning

**What IS statistically significant:**
- Patoshi pattern exists (P < 10^-50 for the pattern itself)

**What is NOT statistically significant when properly analyzed:**
- CFB = Satoshi connection (no independent test)
