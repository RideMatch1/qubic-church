## Technical Deep Dive

### The Derivation Methods

**Method 1: SHA-256 (Bitcoin Native)**
```python
def derive_sha256(qubic_seed):
    hash = sha256(qubic_seed)
    private_key = hash.digest()
    public_key = secp256k1_pubkey(private_key)
    bitcoin_address = pubkey_to_p2pkh(public_key)
    return bitcoin_address, private_key
```

**Method 2: K12 (Keccak-based)**
```python
def derive_k12(qubic_seed):
    hash = k12(qubic_seed, output_length=32)
    private_key = hash
    public_key = secp256k1_pubkey(private_key)
    bitcoin_address = pubkey_to_p2pkh(public_key)
    return bitcoin_address, private_key
```

**Method 3: Qubic (Ternary Hash)**
```python
def derive_qubic(qubic_seed):
    # Qubic's native ternary hash function
    hash = qubic_hash(qubic_seed)
    private_key = ternary_to_binary(hash)
    public_key = secp256k1_pubkey(private_key)
    bitcoin_address = pubkey_to_p2pkh(public_key)
    return bitcoin_address, private_key
```

**Why 3 methods?**
- Redundancy (if one breaks, others work)
- Validation (all 3 should map to same Qubic identity)
- Evolution (test different hash functions)
