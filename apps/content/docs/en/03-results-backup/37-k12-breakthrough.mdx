# The K12 Breakthrough

## Executive Summary

On January 10, 2026, we discovered that we had been using the **wrong hash function** for 3.8 million previous tests. The official Qubic method uses **KangarooTwelve (K12)** hash function applied twice, not SHA256 or Curl as we had assumed.

This discovery came from analyzing the official Qubic source code and fundamentally changed our approach to finding the 1CFB address.

## The Discovery

### What We Were Doing Wrong

For months, we tested seed-to-address generation using:
- **SHA256** hashing: 3,797,590 combinations tested
- **Curl** hashing (IOTA): ~100,000 combinations tested
- Various transformations and XOR operations

**Result**: Found 774 addresses with special properties, but NOT 1CFB.

### The Breakthrough Moment

While researching additional seed batches in the Qubic repository, we found the official documentation:

<Callout type="info" title="Official Qubic Documentation">
From [Qubic Crypto Blog](https://qubic.org/blog-detail/qubic-crypto-details):

> "a 55 lowercase alpha seed is used, but this is mapped to binary values 0 to 25 and then **K12 hashed twice** to make the actual 256 bit privatekey."
</Callout>

## What is KangarooTwelve?

**KangarooTwelve (K12)** is a modern cryptographic hash function:

- **Family**: Keccak-based (same team as SHA-3)
- **Performance**: Faster than SHA256
- **Security**: 128-bit security level
- **Design**: Optimized for modern processors
- **Status**: Designed by the Keccak team (Guido Bertoni, Joan Daemen, et al.)

### Why K12 Instead of SHA256?

1. **Speed**: K12 is significantly faster than SHA256
2. **Qubic Design**: Part of Qubic's overall architecture
3. **Modern**: More recent design (2016) vs SHA256 (2001)
4. **Efficiency**: Better performance on modern hardware

## The Official Method

### Step-by-Step Process

```python
def qubic_seed_to_private_key(seed: str) -> bytes:
    """Official Qubic K12(K12()) method"""

    # Step 1: Validate seed
    assert len(seed) == 55
    assert all('a' <= c <= 'z' for c in seed)

    # Step 2: Convert seed to binary (a-z → 0-25)
    seed_bytes = bytes([ord(c) - ord('a') for c in seed])

    # Step 3: First K12 hash (seed → subseed)
    k12_round1 = KangarooTwelve.new()
    k12_round1.update(seed_bytes)
    subseed = k12_round1.read(32)  # 32 bytes

    # Step 4: Second K12 hash (subseed → private key)
    k12_round2 = KangarooTwelve.new()
    k12_round2.update(subseed)
    private_key = k12_round2.read(32)  # 32 bytes

    return private_key
```

## Test Results

### Pure K12 Method Test

We tested all 23,765 Qubic seeds using the official K12(K12()) method:

**Results**:
- **Seeds tested**: 23,765
- **1CFB found**: No
- **Special addresses found**: 16 addresses with byte sum 2299

**Conclusion**: K12 method works (16 special addresses prove it), but 1CFB requires additional transformations or is outside these seeds.

### K12 + Transformations Test

We then tested K12 combined with known transformation methods:

**Test Parameters**:
- Seeds: 23,765
- Transformation methods: 6 (step7, step13, step19, step27, step33, step121)
- XOR variants: 8 (0, 7, 11, 13, 19, 27, 33, 121)
- **Total combinations**: 1,140,720

**Results**:
- **1CFB found**: No
- **Special addresses found**: 1,045 addresses with byte sum 2299
- **Methods that work**: All combinations produced valid addresses

**Key finding**: Multiple K12-based methods can produce addresses with the special property (byte sum = 2299 = 121 × 19).

## Technical Details

### K12 Implementation

We used the `pycryptodome` library:

```python
from Crypto.Hash import KangarooTwelve

# First round
k12_1 = KangarooTwelve.new()
k12_1.update(seed_bytes)
subseed = k12_1.read(32)

# Second round
k12_2 = KangarooTwelve.new()
k12_2.update(subseed)
private_key = k12_2.read(32)
```

### Performance

K12 is significantly faster than SHA256:
- **K12**: ~13 cycles per byte
- **SHA256**: ~22 cycles per byte
- **Speedup**: ~1.7x faster

This allowed us to test 1.14 million combinations efficiently.
