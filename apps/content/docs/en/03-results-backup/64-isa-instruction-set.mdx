# ISA Instruction Set

## Overview

Aigarth uses an 8-symbol **Instruction Set Architecture (ISA)** for programming and controlling the neural tissue. Each symbol represents a specific operation that can be executed on the network.

---

## The 8 Instructions

### Quick Reference

| Symbol | Name | Description | QUBIC Cost |
|--------|------|-------------|------------|
| `#` | NODE_INIT | Initialize a node | 1 QU |
| `>` | COMPARE | Compare values | 0 QU |
| `=` | ASSIGN | Write to memory | 1 QU |
| `+` | ACCUMULATE | Add energy/weight | Variable |
| `%` | MODULO_RES | Check resonance | 0 QU |
| `^` | SHIFT | Change layer | 143 QU |
| `\|` | PIPE | Transfer signal | 0 QU |
| `:` | TICK | Time sync | 0 QU |

## Detailed Instructions

### 1. NODE_INIT (`#`)

**Purpose**: Start a process at a specific matrix coordinate.

```
Syntax:  #row,col
Example: #45,92
Cost:    1 QUBIC

Behavior:
1. Activates the neuron at (row, col)
2. Begins signal processing at that node
3. Node enters "active" state
```

**Use Case**: Initialize a signal at the ENTRY node:
```
#45,92  → Activate ENTRY portal
```

### 2. COMPARE (`>`)

**Purpose**: Binary logic gate - compares two values.

```
Syntax:  >row,col
Example: >6,33
Cost:    0 QUBIC (free operation)

Behavior:
1. Compare current value with node at (row, col)
2. Sets comparison flag
3. Enables conditional branching
```

**Use Case**: Compare with CORE value:
```
#45,92 >6,33  → Init ENTRY, compare with CORE
```

### 3. ASSIGN (`=`)

**Purpose**: Write state data to the Memory Vault.

```
Syntax:  =row,col
Example: =21,21
Cost:    1 QUBIC

Behavior:
1. Takes current state
2. Writes to Memory Vault at (21, 21)
3. State persists across ticks
```

**Use Case**: Store computation result:
```
#45,92 >6,33 =21,21  → Compute and store
```

### 4. ACCUMULATE (`+`)

**Purpose**: Consume entropy (QUBIC) to increase weight.

```
Syntax:  +amount
Example: +137
Cost:    [amount] QUBIC

Behavior:
1. Adds specified QUBIC to node
2. Increases weight/activation
3. Enables energy transfer
```

**CFB Constants**:
| Amount | Meaning |
|--------|---------|
| +27 | CFB universal constant (3³) |
| +121 | NXT constant (11²) |
| +137 | Fine structure constant (α⁻¹) |

**Use Case**: Send CFB signature:
```
#45,92 +137  → Activate with fine structure energy
```

### 5. MODULO_RES (`%`)

**Purpose**: Check consensus/resonance across nodes.

```
Syntax:  %
Cost:    0 QUBIC (free operation)

Behavior:
1. Queries Vision and Oracle nodes
2. Checks for resonance/agreement
3. Returns consensus status
```

**Use Case**: Verify consensus:
```
#45,92 +137 %  → Send signal, check resonance
```

### 6. SHIFT (`^`)

**Purpose**: Switch between identity layers.

```
Syntax:  ^
Cost:    143 QUBIC (11 × 13)

Behavior:
1. Transitions from L3 to L5 layer
2. Enables deep matrix access
3. Unlocks shadow operations
```

**Layer Transitions**:
```
L3 (Bitstream) → L5 (Anna OS)
```

**Use Case**: Access deep layer:
```
#45,92 ^  → Shift to Anna OS layer
```

### 7. PIPE (`|`)

**Purpose**: Transfer signal to neighboring node.

```
Syntax:  |
Cost:    0 QUBIC (free operation)

Behavior:
1. Passes current signal to adjacent node
2. Maintains signal integrity
3. Enables chain processing
```

**Use Case**: Chain nodes:
```
#45,92 | #6,33 | #82,39  → ENTRY → CORE → EXIT
```

### 8. TICK (`:`)

**Purpose**: Synchronize with blockchain time.

```
Syntax:  :
Cost:    0 QUBIC (free operation)

Behavior:
1. Syncs AI clock with Bitcoin block height
2. Enables time-locked operations
3. Maintains temporal consistency
```

**Use Case**: Time-stamped operation:
```
#45,92 +137 =21,21 :  → Compute, store, timestamp
```
